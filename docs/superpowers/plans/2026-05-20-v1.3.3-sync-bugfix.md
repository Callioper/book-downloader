### Task 1: Fix search_engine.py — field_map, connection reuse, close_all

- [ ] **Step 2: Keep SQLite connections open across calls**

In `backend/search_engine.py`, the `_get_connection()` function (or the search methods) call `.close()` on connections after each query. This is inefficient but more importantly, the `ThreadPoolExecutor` manages the connection lifecycle — closing from the wrong thread causes race conditions.

Remove the `.close()` call at the end of `_get_connection()` (or wherever the connection is closed after search). Instead, connections should remain open and be closed only by `close_all()`.

If the pattern is:
```python
conn = sqlite3.connect(db_path)
conn.row_factory = sqlite3.Row
# ... use conn ...
conn.close()  # REMOVE THIS
```

Remove the `conn.close()` at the end of that method.

- [ ] **Step 3: Ensure `close_all()` method exists and is called**

The `close_all()` method should iterate over all cached connections and close them. Verify it exists in the `SearchEngine` or `ConnectionPool` class. If not, add it:

```python
def close_all(self):
    """Close all cached database connections."""
    for conn in self._connections.values():
        try:
            conn.close()
        except Exception:
            pass
    self._connections.clear()
```

And ensure it's called in the FastAPI shutdown event (in `main.py`).

- [ ] **Step 4: Verify fix**

Run: `python -c "from backend.search_engine import SearchEngine; print('import ok')"`

### Task 7: Fix pdf_bw_compress.py — DCTDecode fallback + Image.open resilience

- [ ] **Step 1: Add try/except for DCTDecode image reading**

Local uses `obj.read_raw_bytes()` directly (line 55). Upstream tries `obj.read_bytes()` first, falling back to `read_raw_bytes()` on `PdfError` (which occurs on DCTDecode streams). Replace:

```python
# BEFORE (local, line ~55):
raw = obj.read_raw_bytes()
img = Image.open(io.BytesIO(raw))

# AFTER (match upstream):
try:
    data = obj.read_bytes()
except pikepdf.PdfError:
    data = obj.read_raw_bytes()  # DCTDecode cannot be decoded by pikepdf
img = None
try:
    img = Image.open(io.BytesIO(data))
except Exception:
    w = int(obj.Width)
    h = int(obj.Height)
    cs = str(obj.get(pikepdf.Name.ColorSpace, '/DeviceRGB'))
    bpc = int(obj.get(pikepdf.Name.BitsPerComponent, 8))
    if '/DeviceGray' in cs:
        mode = '1' if bpc == 1 else 'L'
    elif '/DeviceRGB' in cs:
        mode = 'RGB'
    elif '/DeviceCMYK' in cs:
        mode = 'CMYK'
    else:
        raise ValueError(f"Unsupported colorspace: {cs}")
    img = Image.frombytes(mode, (w, h), data)
```

Add a new endpoint that mirrors the upstream version exactly:

```python
@router.get("/{task_id}/download")
async def download_pdf(task_id: str, type: str = Query(..., description="original|ocr|compressed")):
    task = task_store.get(task_id)
    if not task:
        raise HTTPException(status_code=404, detail="Task not found")
    report = task.get("report", {})
    path_map = {
        "original": report.get("original_path") or report.get("pdf_path"),
        "ocr": report.get("ocr_path"),
        "compressed": report.get("compressed_path"),
    }
    file_path = path_map.get(type)
    if not file_path or not os.path.exists(file_path):
        raise HTTPException(status_code=404, detail="File not found")
    filename = os.path.basename(file_path)

    return FileResponse(
        file_path,
        media_type="application/pdf",
        filename=filename,
    )
```
